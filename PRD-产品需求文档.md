## 二、产品概述

### 2.1 产品目标

为用户提供基于**语义检索**与**相似图片检索**的本地/私有图片管理能力。用户可将存储的图片进行结构化信息录入与标注，并通过自然语言或“以图搜图”快速找到目标图片。

### 2.2 核心价值

- **语义查找**：用自然语言描述“要找什么”，系统理解意图并返回相关图片。
- **相似图查找**：上传或选择一张图，找出图库中视觉/语义上相似的图片。
- **可标注、可筛选**：每张图具备统一元数据（语义、风格、背景、时间等），并支持用户自定义语句/标签，便于后续搜索与筛选。

---

## 三、技术架构约束

- **语言与框架**：Python
- **LLM/检索编排**：LangChain 1.0
- **工作流与状态**：LangGraph
- **智能决策与多步推理**：Agent

系统设计需在上述技术栈下实现“入库 → 解析/标注 → 索引 → 语义/相似检索”的完整链路。

---

## 四、数据模型与存储要求

### 4.1 单张图片必须包含的信息

| 字段 | 说明 | 来源/生成方式 |
|------|------|----------------|
| **语义** | 对图片内容的自然语言描述（场景、主体、动作等） | 自动解析（如多模态模型）或用户补充 |
| **图片** | 图片文件本身或可访问的存储路径/URL | 用户上传或批量导入 |
| **图片风格** | 如：写实、插画、水彩、黑白、复古等 | 自动识别或用户选择/填写 |
| **背景** | 场景背景描述（室内/户外、城市/自然等） | 自动解析或用户标注 |
| **时间** | 拍摄/创建或导入时间 | 系统自动记录或从 EXIF/用户输入 |

### 4.2 用户可扩展信息

- **用户自定义语句/标签**：用户可为每张图片添加一句或多句**特定描述**（如“2024 年团建合影”“产品主图-夏季款”），便于用口语化、业务化表达进行搜索。
- 该信息需参与**语义检索**的匹配与排序。

### 4.3 存储与索引要求

- 语义、风格、背景、时间、用户标签等需**可被检索与过滤**。
- 为支持相似图检索，需存储或可计算**图像向量/特征**（如通过多模态嵌入模型）。
- 建议在 PRD 阶段明确：图片与向量是否分离存储（如对象存储 + 向量库），以便后续扩展。

## 五、功能需求

### 5.1 图片入库与元数据

- **单张/批量上传**：支持用户上传一张或多张图片。
- **自动解析**：入库时自动生成或补充：语义描述、图片风格、背景、时间（若 EXIF 可用则解析）。
- **用户标注**：用户可为每张图片编辑/添加“特定语句信息”（自定义标签或描述），并支持多条。
- **元数据编辑**：用户可事后修改语义、风格、背景、时间、自定义语句等，修改后需触发索引更新。

### 5.2 语义查找

- 用户通过**自然语言输入**描述要找的图片（如“海边日落”“去年聚餐的合影”“所有产品主图”）。
- 系统结合：  
  - 图片的语义、风格、背景、时间、用户自定义语句；  
  - 以及向量检索/LLM 理解；  
  返回与描述最相关的图片列表，并支持按相关度排序。
- 可选：支持**过滤条件**（如时间范围、风格、是否含某标签）与语义组合使用。

### 5.3 相似图片查找

- 用户通过**上传一张图片**或**从图库中选择一张图片**作为查询图。
- 系统基于图像特征/向量，在图库中检索视觉或语义相似的图片，并返回相似度排序结果。
- 可选：支持设置“相似度阈值”或“返回数量上限”。

### 5.4 检索结果的展示与操作

- 以列表或网格形式展示检索结果，展示缩略图及关键元数据（语义摘要、风格、时间、用户标签等）。
- 支持点击单张图片查看大图及完整元数据、用户语句。
- 支持从结果中“用这张图做相似检索”（以图搜图）。

### 5.5 Agent 与工作流（LangGraph）

- 使用 **Agent** 处理复杂查询：例如用户一次输入中既包含语义条件又包含“和某张图类似”的意图，由 Agent 拆解为语义检索 + 相似图检索，再汇总结果。
- 使用 **LangGraph** 编排多步流程：如“解析用户意图 → 选择检索策略（语义/相似/组合）→ 执行检索 → 后处理与排序 → 返回结果”。
- 支持**多轮对话**（可选）：用户可在前一次检索结果基础上追问“只要有人物的”“再排除去年的”等，由 Agent 理解上下文并细化检索。

## 六、非功能需求与扩展需求

### 6.1 性能与规模

- 支持图库规模从数百张到数万张的检索响应时间在可接受范围内（如语义/相似检索主要操作在 3–10 秒内返回）。
- 批量入库时支持队列或异步处理，不阻塞用户操作；大量图片时支持进度展示。

### 6.2 准确性与可解释性

- 语义检索结果应尽量与用户意图一致，避免无关图片排在前面。
- 相似图检索应兼顾视觉相似与语义相似（视产品侧重可调）。
- 可选：对“为什么这张图被召回”提供简单解释（如高亮匹配的标签或语义片段）。

### 6.3 安全与隐私

- 图片及元数据默认存储在用户可控环境（本地或私有云），不依赖将原图上传至公网服务（若使用外部 API，需明确仅发送嵌入/特征或脱敏描述）。
- 若支持多用户，需做权限与数据隔离（按用户/图库维度）。

### 6.4 可维护性与扩展

- 元数据字段（语义、风格、背景、时间、用户语句）在数据模型上便于后续新增字段或新检索维度。
- Agent 与 LangGraph 流程便于增加新的检索策略或第三方工具（如调用外部图搜 API）。

### 6.5 用户体验

- 首次使用或图库为空时，有明确引导：如何上传、如何写语义/标签、如何做语义检索与相似检索。
- 检索无结果时，给出友好提示及建议（如放宽关键词、检查时间范围等）。

### 6.6 导出与备份

- 支持将图库元数据（含用户自定义语句）导出为结构化格式（如 JSON/CSV），便于备份或迁移。
- 可选：支持按检索结果批量导出图片或元数据。

### 6.7 搜索历史与常用查询（可选）

- 记录用户最近的语义查询与相似图查询，便于一键再次执行或微调。
- 支持将某次检索条件保存为“常用搜索”或“智能相册”规则，便于后续一键刷新结果。

## 七、成功标准（验收方向）

- 用户能完成：上传图片 → 自动/手动补充语义、风格、背景、时间及自定义语句 → 通过语义查找与相似图查找稳定找到目标图片。
- 在多轮或组合意图下，Agent + LangGraph 能正确拆解并执行语义+相似图检索，并返回合理结果。
- 数据模型与存储满足：语义、图片、图片风格、背景、时间、用户标记语句均落库并可检索；技术栈符合 Python + LangChain 1.0 + LangGraph + Agent。
